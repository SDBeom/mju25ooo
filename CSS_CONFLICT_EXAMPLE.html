✅ 예시 1: Media Query 방식 (충돌 없음)

창 크기를 조절해보세요!

  • 📱 < 768px: 빨강
  • 📱 768px ~ 1023px: 주황
  • 💻 ≥ 1024px: 초록

✅ 예시 2: 클래스 방식 (충돌 없음)

Desktop 클래스:

이 박스는 box--desktop 클래스가 있습니다

Tablet 클래스:

이 박스는 box--tablet 클래스가 있습니다

Mobile 클래스:

이 박스는 box--mobile 클래스가 있습니다

💡 각 박스는 다른 클래스를 가지고 있어서 충돌하지 않습니다!

❌ 예시 3: 잘못된 방식 (충돌 발생!)

항상 초록색만 나옵니다

⚠️ 같은 선택자를 여러 번 정의하면 마지막 것만 적용됩니다.
이렇게 하면 안 됩니다!

📚 핵심 정리

✅ 충돌하지 않는 이유:

  1. Media Query: 조건이 다름 (화면 크기별로 다른 조건)
  2. 클래스 방식: Specificity가 다름 (선택자의 구체성이 다름)

❌ 충돌하는 경우:

  1. 같은 선택자를 조건 없이 여러 번 정의
  2. 같은 Specificity에서 같은 속성을 여러 번 정의

💡 우리 프로젝트에서는:

JSX 파일 1개 + CSS 파일 1개로 충분합니다!
Media Query나 클래스로 분기하면 충돌 없이 깔끔하게 처리됩니다.
